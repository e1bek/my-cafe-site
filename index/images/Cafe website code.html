ChatGPT said:

They’re actually showing — but only if your browser can load them from the Unsplash links I put in the code.

Right now, your HTML points to dynamic image URLs like:

html
<img src="https://source.unsplash.com/400x300/?coffee" alt="Coffee">

That means:

  • Every time the page loads, it fetches a random image from Unsplash with those keywords.

  • If you’re offline, behind a slow connection, or Unsplash is blocked in your region, you’ll see no image (or a broken icon).

  • Some browsers block “mixed content” if you’re running the HTML locally and not through a server.

Solutions:

  1. Use fixed image URLs instead of random Unsplash queries
    Example:

    html
    <img src="images/coffee.jpg" alt="Coffee">

    Then put coffee.jpg in a local images folder next to your HTML file.

  2. Download Unsplash images and host them locally

    • Go to Unsplash.com, search for coffee, latte, etc.

    • Download the image and put it into your project’s folder.

    • Update the <img> src path to your local file.

  3. Run a local web server
    If you open the file by double-clicking it, some browsers block certain resources.

    • If you have Python, you can do:

      bash
      python -m http.server

      and open http://localhost:8000 to view your site.
